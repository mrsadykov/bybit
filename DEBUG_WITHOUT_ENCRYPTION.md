# Отладка без шифрования API Secret

## Проблема

Если вы получаете ошибку 401 при проверке баланса, возможно проблема в шифровании/расшифровке `api_secret`.

## Решение: Временное отключение шифрования

Для отладки можно создать аккаунт **без шифрования** `api_secret`:

```bash
php artisan create-bybit-account --force --no-encrypt
```

⚠️ **ВНИМАНИЕ**: Это только для отладки! В production всегда используйте шифрование!

## Что делает `--no-encrypt`

1. Удаляет старые аккаунты (если есть)
2. Сохраняет `api_secret` в БД **без шифрования**
3. Позволяет проверить, не в шифровании ли проблема

## После отладки

После того как найдете проблему, **обязательно** пересоздайте аккаунты с шифрованием:

```bash
php artisan create-bybit-account --force
```

## Проверка

После создания аккаунта без шифрования проверьте:

```bash
php artisan api:test --account=ID
```

## Важно

- `--no-encrypt` сохраняет секреты в открытом виде в БД
- Используйте только для отладки на локальной машине
- **Никогда** не используйте в production!
- После отладки удалите аккаунты и создайте заново с шифрованием
